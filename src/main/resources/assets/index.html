In-browser examples

These are C++ examples compiled to Emscripten. All the packing happens automatically in CMake scripts, no extra work, same process as creating iOS applications, Snap packages, AppImages and Universal Windows Platform containers.

These are simple code tests, and don't have much in the way of interactivity.

  • Environment information: Prints storage paths in the virtual filesystem provided by Emscripten
  • Basic OpenGL: A basic OpenGL scene with a faulty emulation of 2D texture arrays
  • Advanced scene: A scene built using the OpenGL abstraction and a custom content pipeline.
    The pipeline is found in `hbirchtree/coffeecutie/tools/data-management/pressure-cooker`. It has several tasks:
    • Shaders are cross-compiled into multiple variants, such as ESSL 1.00, ESSL 3.00, GLSL 3.30 and GLSL 4.60.
    • Textures are compressed into different formats. For now it supports DXTn/S3TC. At the same time this process generates mipmaps to avoid the runtime costs of mipmapping.
    • Meshes are processed into raw buffers for direct upload into OpenGL, with added compression.
    • Finally the files are put into a virtual filesystem for memory-mapped access, enhancing the possibilities for content streaming.
    In this particular case, support for OpenGL ES 2.0 is not fully implemented, meaning there are no textures.
    NOTE: This one opens in a new tab for the sake of input focus.

Statistics

These numbers are collected from devices sending runtime reports to `coffee.birchtrees.me`. This is made possible by an embedded profiler in the application, activated by specifying the following environment variable:

COFFEE_REPORT_URL=https://<server>/<endpoint> <program>

or on Android builds, this is done by specifying an activity intent:

adb shell am start -n <com.package/.Activity> --es COFFEE_REPORT_URL <report-server>
